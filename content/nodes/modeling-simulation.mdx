# 모델링 & 시뮬레이션

게임 세계의 상태 정의와 업데이트. 게임 엔진이 어떻게 시뮬레이션을 실행하는지 이해하는 것이 모든 게임 개발의 기초입니다.

## 핵심 개념

- 게임 루프
- 틱/프레임
- 물리 스텝
- 상태 머신
- ECS 패턴
- 결정론적 시뮬레이션

## 활용 분야

- 모든 게임
- RTS (스타크래프트)
- MOBA (LoL)
- 격투 게임

## 게임 루프와 틱

게임 엔진의 핵심은 게임 루프입니다. 매 틱(또는 프레임)마다 입력 처리부터 렌더링까지 모든 작업이 순차적으로 실행됩니다.

### 한 틱 안에서 일어나는 일

게임 엔진은 매 틱마다 다음 작업들을 순서대로 처리합니다. 모든 작업이 틱 시간(예: 16.67ms for 60fps) 안에 완료되어야 합니다.

#### 1. 입력 처리 (Input)

키보드, 마우스, 게임패드 입력을 수집하고 처리합니다. 네트워크 게임에서는 원격 플레이어의 입력도 이 단계에서 처리됩니다.

#### 2. 게임 로직 (Update)

AI, 스킬 쿨다운, 버프/디버프, 스폰 등 게임 규칙을 실행합니다. 이 단계에서 게임 상태가 변경됩니다.

#### 3. 물리 시뮬레이션 (Physics)

충돌 감지, 강체 역학, 캐릭터 컨트롤러를 처리합니다. 물리는 별도의 고정 타임스텝으로 실행될 수 있습니다.

#### 4. 네트워크 (Network)

상태 동기화, 패킷 전송/수신, RPC 호출을 처리합니다. 클라이언트-서버 모델에서는 이 단계가 특히 중요합니다.

#### 5. 렌더링 준비 (Late Update)

카메라 추적, UI 갱신, 애니메이션 블렌딩 등 렌더링 직전 작업을 수행합니다.

#### 6. 렌더링 (Render)

화면에 그립니다. GPU에 드로우콜을 보내고 프레임 버퍼를 스왑합니다.
